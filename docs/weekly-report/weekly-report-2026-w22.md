# WonderLens Activity AutoDesign — 周报 W22（2026-05-23 ~ 2026-05-29）

## 下游运行时转换与端到端验证

- 打通活动包到 fullstack-demo 和 WonderLens AI 的验证链路，使新生成包可以被导入、转换、加载、播放或按支持状态诚实阻断。
- 完成覆盖简单对话、拍照采集、判断降级、复杂界面阻断和事实资产的端到端子集验证，证明新包已能跨消费端跑通关键路径。
- 强化运行时回合契约，使 `Runtime AI instruction` 必须包含目标、约束、情绪、分支、进展证据和屏幕状态，降低下游生成 `step_instructions` 的猜测空间。
- 修复下游验证暴露的导入和加载差异，使丰富运行时回合能转换为可执行步骤，并让签名字段兼容当前活动包。

## Demo 包契约与支持状态

- 上线面向 demo 消费端的活动包契约，用 `demo_support` 和 `asset_manifest` 明确可玩性、实体绑定、素材需求和导入预期。
- 收敛支持、降级和不支持三类状态，使排序、绘画、证书等当前 UI 无法承载的机制不会被伪装成可玩活动。
- 明确 Cat5 采集类资产规则，让屏幕卡片可以引导孩子行动，同时保留真实拍照采集和 `photo_id` 交接语义。
- 加固事实和参考类素材要求，要求星座、地图、艺术品等内容绑定可信来源，避免运行时使用任意生成近似图。

## 运行时素材生成与审查

- 实现 `asset_build=generate_and_curate` 流程，把素材从提示和审核线索推进到包内 PNG、来源元数据、清单路径和审查仪表盘。
- 新增资产输出验证，覆盖图片尺寸、包内路径、来源哈希、来源许可和绝对路径污染，减少可审但不可用的素材产物。
- 完善评审仪表盘的素材状态展示，使生成图、参考来源、缺失项、回退状态和工作项能被审查者直接看到。
- 跑通真实素材冒烟验证，验证插画生成、参考来源接收、变体生成和资产验证可以在同一工作流内闭环。

## 源意图审计与高风险玩法收敛

- 强化工作簿证据要求，使源意图审计必须回到原始输入行取证，而不是只依赖规范化快照或适配摘要。
- 修复星座数量活动的意图偏移，把玩法收回到先选数字、再揭示真实星座、再补充背景知识的原始承诺。
- 加固多个易漂移活动的源意图锁，降低职业决策、植物部位、颜色艺术、证书制作和词汇猜测等玩法被弱化的风险。
- 规划下一轮源意图试跑，锁定高风险和对照样本，用独立审计、运行时指令审查、素材审查和双消费端验证决定能否放大跑批。

## 质量修复与代码清理

- 优化源对照矩阵和审查验证，使缺失工作簿证据、薄弱运行时契约、过度泛化分支和陈旧审查页更早暴露。
- 完善计划和目标索引状态，让已完成、计划中和待执行的运行时转换、demo 契约、资产管线、E2E 验证和试跑工作可以被连续追踪。

## 下周关注点

- 推进源意图试跑的实际生成和审查，优先验证星座、植物部位、职业决策、局部揭示和轻量对照活动。
- 继续跟进参考类素材的来源接收和许可判断，确保事实资产在消费端展示前已有可审证据。
- 对齐 fullstack-demo 和 WonderLens AI 的活动导入契约，避免试跑通过作者审查后在下游转换中丢失源意图。
- 根据试跑报告决定是否进入更大范围或全量跑批，并把未解决的产品和运行时能力缺口转成明确后续项。
