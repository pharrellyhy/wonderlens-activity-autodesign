# WonderLens Activity AutoDesign — 周报 W26（2026-06-20 ~ 2026-06-26）

## Demo 就绪活动包刷新

- 完成旧版 demo 包转换，使早期活动也具备 demo 支持状态、实体兼容性、参数化语义和包内素材声明。
- 补齐核心活动集的运行时素材，使 demo 就绪包可以直接引用本地场景、回合图、物件卡和合成画面。
- 收敛 Polka Dot Patrol 和 Dandelion 的 Cat5 目录玩法，把原本依赖任意拍照或触觉判断的流程改成可验证的属性选择。
- 刷新活动覆盖说明，使规范包、demo 就绪包和历史旧版输出之间的边界更清楚。

## Activity Display Contract V1

- 建立 `activity_display_contract_v1` 生产者契约，使每个运行帧都能声明布局、控件、显示素材、验证策略和反馈效果。
- 回填当前核心包和 WonderLens AI 种子包的显示契约，使下游不再需要从活动名或说明文字推断屏幕行为。
- 新增显示契约 schema、validator 和 fixture 覆盖，使未知布局、未绑定素材、缺失验证目标和非英文反馈字段可以被自动拦截。
- 明确显示契约不进入 tag block 或 demo support，降低匹配元数据、demo 导入和 app 显示运行时之间的职责耦合。

## 运行时验证与对话收敛

- 更新四个运行时验证包的响应验证语义，使首字母采集、情绪线索、局部揭示和动物声音都能从现有包元数据完成判断。
- 移除四个包的独立开场桥接，使运行流程从只讲规则的步骤开始，并把首次孩子动作保留到第一回合或第一次采集。
- 收紧四个包的面向孩子桥接文案，使规则不提前发出动作邀请、局部揭示不编造线索、动物声音每轮只验证一个清晰回应。
- 将 Phoneme Treasure Hunt 从固定音素语言收敛到首字母策略，使运行时可以围绕第一张识别照片建立后续采集规则。

## 包契约与下游边界

- 引入 Cat5 运行时判断 demo 模板，使需要运行时判断的活动可以被标记为降级支持而不是伪装成完全支持。
- 保留不依赖显示契约的响应验证路径，使 WonderLens AI 可以先消费现有包形状，同时为后续显示契约运行时留出迁移空间。
- 强化局部揭示的隐藏答案边界，使内部答案元数据可以服务验证，但面向孩子的运行文本不会提前泄露目标。

## 质量修复与代码清理

- 清理过时和未纳入规范集合的旧活动包，降低审查者在当前活动集里误选陈旧产物的风险。
- 修复四个包的对话桥接细节，使实测运行时暴露的聊天化规则、线索漂移和首轮动作时机问题回到包源头处理。
- 更新计划和目标索引，使显示契约、四包响应验证、只讲规则改造和后续素材候选工作可以被连续追踪。

## 下周关注点

- 推进 WonderLens AI 对 `activity_display_contract_v1` 的运行时消费和 Activity WebSocket 验证，闭环生产者合同到下游执行。
- 继续验证四个响应验证包在 WonderLens AI 中的实际判断行为，确保包源文本和运行时镜像不再漂移。
- 推进 Gemini 多候选和 curator 接受流程，避免单张低质量插画输出直接进入正式构建入口。
- 评估新的 demo 就绪规范包是否足以支撑下一轮全量流程试跑，并优先处理仍处于降级支持的运行时判断能力缺口。
