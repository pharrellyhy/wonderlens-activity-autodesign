# WonderLens Activity AutoDesign — 周报 W23（2026-05-30 ~ 2026-06-05）

## 源意图与全量管线契约

- 上线全量 agentic 管线契约，把源意图审计、包生成、素材阶段、消费端验证、对话 QA、图像审查和最终复核串成可重复的运行流程。
- 完成源意图试跑并修复漂移活动，使新包能同时保留工作簿承诺、运行时指令和审查证据。
- 建立全量运行目标和执行前置条件，使后续大批量生成可以从明确队列、角色分工、消费端目标和重试策略启动。
- 简化核心运行文档和任务队列，使默认 operator 流程回到 active-only 的入口，减少历史批次和长篇说明对新任务的干扰。

## 素材生成与图像 QA

- 实现 `asset_build=generate_and_curate` 的全量素材闭环，把活动素材推进到包内图片、来源元数据、质量记录和审查仪表盘。
- 加固图像素材闸门，使占位感、过早揭示答案、场景复用、选择物混入背景和伪 UI 元素能在生成后被明确拦截。
- 修复多个活动的场景和物件素材，使重排、识别、诗歌、访谈、音素和绘画类包更贴近各自玩法节奏。
- 收敛 fullstack-style 素材包规则，使每个可玩包都围绕标准回合场景、必要物件和 Cat5 合成画面声明素材需求。
- 暂停质量不达标的全量图片继续生成，并把剩余素材按来源敏感度、选择物风险、揭示节奏和动作保真度分组，便于恢复时优先处理。

## 子集验证与消费端证据

- 拆出干净的子集工作流门禁，把可复用脚本、测试和文档从未完成的全量运行产物中剥离出来。
- 完成高风险子集验证，使星座、绘画、音素、识别和采集活动首次按当前全量闸门跑通生成、素材、审查和双消费端证据。
- 修复音素子集验证中的画面泛化问题，使故事场景、物件候选、发音证据和采集合成更符合源意图。
- 记录 fullstack-demo 对 Cat5 单项采集的下游限制，使包自身通过后仍能诚实暴露消费端后续项。

## WonderLens AI 运行时准备

- 生成一组 WonderLens AI-ready 活动包，覆盖当前核心活动集的文本包、素材包、模板、支持状态和运行时生成报告。
- 收敛最小 adapter 支持，使降级、可玩和判断类活动在运行时转换时不再依赖过度特殊化的包形状。
- 修复绘画活动的运行时表达，使步骤确认围绕孩子的自我报告和材料流程推进，而不是做未经支持的视觉质量判断。

## 实体参数化包契约

- 新增 producer 侧实体参数化契约，使活动包能明确声明 handoff entity 是固定样例、主题上下文、动态目标、属性来源、初始音来源、词语来源还是素材目录目标。
- 建立当前 runtime-ready 活动的参数化分类矩阵，使审查者和下游运行时不再从活动名、宽泛匹配标签或 prose 中猜测实体绑定语义。
- 加固 demo 包校验和 fixture 覆盖，使动态参数化缺少运行字段、固定包缺少常量声明、音素包硬编码目标音等问题可以被自动拦截。
- 增强 review dashboard 的参数化 verdict，使 reviewer 能直接看到模式、完整性、动态字段、冻结字段、handoff 要求、证据和处理动作。

## 质量修复与代码清理

- 优化全量管线展示页，使工作流动画、步骤详情、文本包结构和素材规则可以被操作员更快理解和复核。
- 移除过时的 recap 图片素材要求，使图片契约回到运行时真正需要的场景、图标、物件和合成画面。
- 整理计划、目标和任务索引状态，使已完成、计划中和待恢复的工作流可以连续追踪。
